  1. Creating Snapshots:

    Initializing a repository

    git init

    Staging files

    add file1.js
    git add file1.js file2.js
    git add *.js
    git add .

    Viewing the status

    git status
    git status -s

    Committing the staged files

    commit -m “Message”
    git commit

    Skipping the staging area

    commit -am “Message”

    Removing files

    git rm file1.js
    git rm --cached file1.js

    Renaming or moving files

    mv file1.js file1.txt

    Viewing the staged/unstaged changes

    git diff
    git diff --staged
    git diff --cached

    Viewing the history

    git log
    git log --oneline
    git log --reverse

    Viewing a commit

    git show 921a2ff
    git show HEAD
    git show HEAD~2
    git show HEAD:file.js

    Unstaging files (undoing git add)

    restore --staged file.js

    Discarding local changes

    git restore file.js
    git restore file1.js file2.js
    git restore .
    git clean -fd

    Restoring an earlier version of a file

    restore --source=HEAD~2 file.js
